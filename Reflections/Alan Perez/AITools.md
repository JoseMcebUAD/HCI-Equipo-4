# Gemini use to produce quality UI

When using Google Gemini to try to produce UI that includes specific features related with usability features, a notable issue was that the first approach is too simple, it has no colors nor details, just text and buttons, the buttons has color but that’s it, <mark>it does achieve its objective of making a functional prototype, but it has a lot of work to polish to reach a decent product as final goal.</mark> 
AI tools excel at logic but lack aesthetic judgment, <mark>AI can build the foundation, however it outputs a generic prototype, because visual design requires human direction</mark>, therefore the quality is good for prototyping, but poor for final products. 

The experience producing UI with Gemini was good, it is easy to use, it just comes down to typing out the requirements and testing the results, even the AI can generate a prompt based on the project’s needs, which can be used to start the UI generation, <mark>from there it's just a matter of making iterative upgrades alongside the AI.</mark> 
The AI tool allows to focus purely on an iterative design loop: testing, refining, and requesting upgrades; making the process continuous. 

Considering the final result of the UI, it included the usability attribute to a midterm extent, the purpose of the buttons was clear enough, is not crystal clear but is not clueless either, so it is on a midpoint. 
<mark>AI can only achieve a midpoint of usability, because it relies on programmed heuristics, not human empathy.</mark> 

The UX role will change significantly in the future with the huge growth of AI tools, the way to produce code and UI is each day more easy and efficient thanks to AI, and this should make the industry progress even faster than it is now, the UX role will be similar but it should include requesting AI tools knowledge to compete with the standard nowadays. <mark>UX pros will spend less time drawing and more time validating the AI's output against real user needs.</mark>
