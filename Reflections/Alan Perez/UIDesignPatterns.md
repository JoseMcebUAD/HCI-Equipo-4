# UI Design Pattern Applied in Team Project Prototype

The selected task for this reflection is the main design of the rescheduling functionality in the ClinicaWeb Agenda Module. This task is already completed, and that does not reduce its value for analysis, because it represents a real solution to one of the clinic’s main problems, which is the lack of synchronization when appointments change. <mark>The important part is that the interface now shows how the system organizes the information to make the process clearer for the user.</mark>

A relevant UI pattern for this task is visual hierarchy and information grouping, because the interface separates the elements into clear sections instead of mixing everything together. In the New Appointment modal, the fields are organized in a structured way, grouping related inputs such as event data, time, and additional information like the payment receipt. <mark>This allows the user to scan the form quickly and understand what each section is for without confusion</mark>, even though all the information is visible at once, the grouping helps reduce cognitive effort by making the layout predictable and easy to follow.

This pattern is relevant to the usability goals of the project because our Persona: Patricia, needs a system that feels simple and fast, similar to her notebook but more reliable, <mark>by organizing the information clearly, the interface improves readability, supports faster decision-making, and reduces the chance of errors in a high-pressure environment,</mark> in that way, the design supports usability not by hiding information, but by structuring it in a way that is easy to process and efficient to use.
