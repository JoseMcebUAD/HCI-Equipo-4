# Difference between UI and UX

<mark>User Interface (UI) are the visual and interactive elements</mark> designed for the user to achieve the goal of the app, to ensure that <mark>the interface needs to be intuitive and clean</mark> from the user point of view, and <mark>the usual metrics</mark> used to know the level of how well done an interface is, <mark>are efficiency, responsiveness and a clean aesthetic</mark>. 

<mark>User Experience (UX) is the user’s overall experience and feeling</mark> got from an app, in order to provide a good user experience, <mark>making the app feel clear, organized, and most importantly, aligned with the user needs, is the main focus, and the usual metrics</mark> used to know the level of how well done an user experience is, <mark>are positive emotional response, user control, and comprehension</mark>.

In the ClinicaWeb Agenda Module, the <mark>UI could be demonstrated by the design of the new appointment modal</mark>, which uses <mark>interactive elements like dropdown menus, a date picker, and a notable blue button</mark> to save the information and ensure visual clarity, readability, and a clean aesthetic. 

The <mark>UX could be demonstrated by an user evaluating pending requests and clinic availability simultaneously</mark>, because the requests and the calendar schedule are placed alongside, in this example <mark>minimizing the cognitive load and keeping main functions visible</mark> without the need to navigate through the app, is what makes the scheduling workflow feel organized, efficient, and aligned with the user’s needs.
